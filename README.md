# たしざんでんしゃ 🚃

4歳からの足し算アプリ。10×10 の数字盤の上を電車が進み、「8 から 7 すすむとどこ?」を
**電車が 10 で折り返して進む**動きで体感しながら、楽しく何度も足し算を反復できる。

公文の数字盤と同じ見え方なので、`+7 = 「1段下がって3つ戻る」`のような盤面上の形を
空間で覚えられる。最初は 30 以下 × 足す数 1〜9 が対象。

## あそびかた

- **はじめてモード**: 「▶ すすむ」を押すと電車が1マスずつ進む。足す数の回数だけ進めばゴール。
- **じょうきゅうモード**: どこに着くか予想して、盤面のマスを直接タップ。
- 正解すると、盤面で動いていた電車が下の「きみのでんしゃ」へスーッと飛んで連結する。
- 10両つながると「しゅっぱつ しんこう！」で編成が走り去り、また機関車 🚂 1両から
  集め直す(`せいかい` の累計はそのまま伸びていく)。
- 右上の ⚙️ から、**足す数の選択(+1〜+9 固定 / おまかせ上限 〜5・〜7・〜9)**、
  **はんい(20 / 30 / 50 / 100まで)**、
  モード切替、音の ON/OFF、編成リセットができる。はんいに応じて数字盤の行数が変わり、
  **100 はちょうど 10×10(公文の数字盤)**になる(列は常に10、10で折り返し)。
- 音声は問題の読み上げ・数え上げ・ほめ言葉をしゃべる(iPad 内蔵音声、オフライン可)。

### チャレンジモード

ヘッダーの 🏆 ボタンから2種類のチャレンジを遊べる(マスをタップして回答、出題はおまかせ混合)。

- **れんぞく せいかい**: 1問でも間違えたら終了。連続正解数を競う。正解するごとに
  **京浜東北線(JK01 大宮 → … → JK47 大船)を1駅ずつ進む路線図**が表示される(駅名・ふりがな付き)。
- **60びょう チャレンジ**: 60秒で何問解けるかを競う(間違いはやりなおし、タイマーは進む)。
  解けた数を**電車のスピード**に換算して表示し、速いほど **ふつう→きゅうこう→とっきゅう→
  しんかんせん** とランクが上がる(1問あたり 難易度〜5で+10 / 〜7で+15 / 〜9で+20 km/h)。
- 開始前に **むずかしさ(おまかせの足す数の上限 〜5 / 〜7 / 〜9)** を選べる。
- チャレンジの正解音は「ピンポーン」(Web Audio で合成、音 ON 時)。

それぞれのベスト記録を保存し、更新時は「しんきろく!」。結果から「もういちど / おわる」。
路線モード中でもそのテーマのままチャレンジできる。

### 隠し「路線モード」

タイトル「たしざんでんしゃ」を **2秒長押し**すると路線を選べる(子どもが偶然開きにくい)。

| 路線 | 駅ナンバー | テーマ色 | 完成両数 |
|------|-----------|----------|:---:|
| 山手線 | JY01〜JY30(東京〜有楽町、30駅) | ウグイス | 11両 |
| 京浜東北線 | JK01〜JK30(大宮〜) | スカイブルー | 10両 |
| 常磐線 | JJ01〜JJ10(上野〜取手)+以北の駅名 | E531系の青+青磁帯 | 15両 |
| 日比谷線 | H01〜H22(虎ノ門ヒルズ H06 含む) | シルバー | 7両 |

- 数字盤の各マスが駅(駅ナンバー＋駅名)になり、上部に電車のいる現在駅を表示。
- カラーテーマと電車の色が路線色に変わり、**完成両数も路線ごと**に変わる(山手は11両で出発、など)。
- 「ノーマル」を選ぶと通常の 🚃・青テーマ・10両に戻る。選択は保存される。

## 動かす

ブラウザのセキュリティ上、ES Module と Service Worker は `file://` では動かないため、
簡易サーバ経由で開く:

```sh
python3 -m http.server 8000
# → ブラウザで http://localhost:8000/index.html を開く
```

### iPad のホーム画面に追加(オフラインで使う)

1. 家の Wi-Fi 環境で、iPad の Safari から上記 URL(同じネットワーク内の PC の IP など)を開く
2. 共有ボタン → **「ホーム画面に追加」**
3. 以降はホーム画面のアイコンから、全画面・オフラインで起動できる

## GitHub Pages で公開

ビルド不要の静的サイトなので、ファイル一式をそのまま GitHub Pages で配信できる
(相対パスで作ってあるので `/calc-app/` のようなサブパスでも動く)。ルートの空ファイル
`.nojekyll` で Jekyll を無効化し、`js/` や `icons/` を含む全ファイルをそのまま配る。

1. コードを GitHub の `main` に push(下記「公開手順」参照)。
2. リポジトリの **Settings → Pages** を開く。
3. **Source: Deploy from a branch** → **Branch: `main` / `(root)`** → Save。
4. 数十秒後、`https://<ユーザー名>.github.io/calc-app/` で公開される
   (このリポジトリなら **https://tkrudagawa.github.io/calc-app/**)。
5. iPad の Safari でそのURLを開き、共有 → **「ホーム画面に追加」**(HTTPS なので PWA も動く)。

> 更新は `main` に push するたびに自動で反映される。`sw.js`(Service Worker)は
> オフライン用に静的ファイルをキャッシュするので、内容を更新したら `sw.js` の
> `CACHE` バージョン(`tashizan-densha-vN`)を上げると確実に最新へ入れ替わる。

## テスト

中核ロジック(盤面の折り返し経路・出題生成・状態遷移)は Node 標準のテストランナーで検証:

```sh
npm test
```

## 構成

| ファイル | 役割 |
|----------|------|
| `index.html` / `styles.css` | 画面の骨格と見た目 |
| `js/board.js` | 数 ↔ (行,列) 変換・折り返し経路・盤面の最大数 `MAX`(純粋関数 / 共通定数) |
| `js/problem.js` | 出題生成(足す数の選択 / おまかせ、純粋関数) |
| `js/lines.js` | 隠し路線モードのデータ(色・駅名/駅ナンバー・完成両数、純粋データ) |
| `js/challenge.js` | チャレンジの状態機械(連続/とけた数・終了判定・ベスト記録、純粋ロジック) |
| `js/game.js` | 状態管理・正誤判定・編成・永続化(localStorage) |
| `js/speech.js` | 音声読み上げ(SpeechSynthesis、助詞「は」は「わ」と発音) |
| `js/ui.js` | 盤面描画・電車アニメ・連結演出・操作 |
| `js/main.js` | 起動と結線 |
| `manifest.webmanifest` / `sw.js` | PWA(ホーム画面追加・オフライン) |
| `scripts/make-icons.mjs` | アイコン生成(ビルド時のみ。`node scripts/make-icons.mjs`) |

設計の経緯は `docs/`（ブレインストーミングの設計書）を参照。
