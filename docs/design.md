# 足し算でんしゃ(Tashizan Densha)— 4歳児向け足し算アプリ 設計プラン

## Context(背景・目的)

4歳の子どもが公文で足し算を学習中だが、「足す7」あたりでつまづいて苦戦している。
原因は、6〜9 のような大きな数を「数え足し」だけで処理しきれなくなること。
本アプリは、**楽しく何度も足し算を反復させ、数の感覚(number sense)を自然に身につけさせる**ことを目的とする。

中心となるメタファーは「**10×10 の数字盤の上を電車が進む**」。
公文でも使う 10 マス区切りの数字盤を使い、「8 から 7 進むとどこに着くか」を、
電車が **10 で折り返して進む** 動きとして体験させる。これにより、
**+7 = 「1段下がって3つ戻る」** のような盤面上の形を空間で覚えられる。

最初の段階は **30 以下の数 × 足す数 1〜9** の習得を目標とする。

## ゴール / 非ゴール

**ゴール**
- iPad の Safari で開け、ホーム画面に追加してオフライン・全画面で遊べる(PWA)
- 1〜30 の3段数字盤上を、電車が 10 で折り返しながら進む
- 足す数を選んで集中練習(+1〜+9)+「おまかせ」でミックス出題
- 初級(すすむボタンで1マスずつ)/上級(ゴールを予想してタップ)をモード切替
- 正解ごとに自分の電車の編成が伸びる(反復の動機づけ)
- 日本語音声で問題読み上げ・数え上げ・ほめ言葉

**非ゴール(YAGNI)**
- ログイン・アカウント・クラウド保存(ローカル完結)
- 引き算・かけ算など他の演算(まずは足し算のみ)
- 複数ユーザー / 進捗の詳細統計ダッシュボード
- App Store 配布・ネイティブ化(Web/PWA で完結)
- 31 以上への自動拡張(設計上は拡張可能にしておくが、初版は 1〜30)

## 技術選定

- **バニラ HTML / CSS / JavaScript**(ビルド不要、依存ゼロ)
  - 理由: ファイルを開けば動く / iPad ホーム画面に置ける手軽さを最優先。1画面アプリにフレームワークは過剰。
- **PWA**: `manifest.webmanifest` + Service Worker(`sw.js`)でオフライン対応・全画面起動
- **音声**: Web 標準の `SpeechSynthesis`(`ja-JP`)。録音ファイル不要・オフライン可
- **効果音(任意)**: WebAudio の簡単なビープ、または無音でも成立する作り
- テスト: ロジック層は純粋関数に切り出し、軽量テスト(後述)

## モジュール構成(責務分離)

各モジュールは「何をするか・どう使うか・何に依存するか」が明確になるよう純粋ロジックと表示を分ける。

| ファイル | 責務 | 依存 |
|----------|------|------|
| `index.html` | 画面の骨格・要素配置 | — |
| `styles.css` | レイアウト・大きなタップ領域・色・アニメーション | — |
| `js/board.js` | 盤面モデル: 数値↔(行,列)変換、折り返し経路の算出 | なし(純粋関数) |
| `js/problem.js` | 出題生成: 足す数の選択/おまかせ、start≤30 制約で問題を作る | なし(純粋関数) |
| `js/speech.js` | 音声読み上げ(問題文・数え上げ・ほめ言葉)のラッパー | SpeechSynthesis |
| `js/game.js` | 状態管理: 現在の問題・電車位置・モード・編成長・正誤判定 | board, problem |
| `js/ui.js` | DOM 描画・イベント(すすむ/マスタップ)・アニメーション・音声呼出 | game, board, speech |
| `js/main.js` | 起動・設定オーバーレイ・各モジュール結線 | 全部 |
| `manifest.webmanifest` | PWA メタ(名前・アイコン・全画面) | — |
| `sw.js` | Service Worker: 静的ファイルをキャッシュしオフライン化 | — |

> 純粋関数(`board.js` / `problem.js`)を分離する狙い: 折り返し経路や出題ロジックは間違えやすい核心部なので、表示と切り離して単体テストしやすくする。

## 中核ロジックの仕様

### 盤面モデル(`board.js`)
- 盤面は 1〜30、10列固定。行=`Math.ceil(n/10)`、列=`((n-1)%10)+1`(標準的な百ます表と同じ左→右・上→下)。
- **進行経路**: start から数を1ずつ増やして goal まで。列10の次は次行の列1へ「折り返す」。
  - 例: 8 →(9,10)→ 折り返し →(11,12,13,14,15)。経路は `[9,10,11,12,13,14,15]` の各マスを順に通過。
- セル中心座標はグリッドの DOM から取得し、電車アニメーションに使う。

### 出題生成(`problem.js`)
- 入力: 選択中の足す数(`addend` または「おまかせ」)、上限(初版は 30)。
- **おまかせ**: addend を 1〜9 からランダム。
- start は `1 ≤ start` かつ `start + addend ≤ 30` を満たす範囲でランダム。
  - これにより goal が必ず盤面内に収まる。
- 返り値: `{ start, addend, goal }`。

### モード(`game.js` + `ui.js`)
- **初級**: 「すすむ」ボタン1回で電車が次のマスへ1つ進む(折り返し含む)。進むたびに音声で「いち、に、…」。addend 回進んで goal に着いたら正解判定→演出。
- **上級**: 盤面のマスを直接タップ。`goal` なら正解、それ以外は不正解フィードバック(電車が首を振る等)してリトライ。

### ごほうび(`game.js`)
- 正解1回ごとに「きみのでんしゃ」へ車両を1両連結(編成長 = 累計正解数)。
- 編成長は `localStorage` に保存し、次回も続きから(リセットは設定から)。

### 音声(`speech.js`)
- 出題時: 「<start> たす <addend> は?」を `ja-JP` で読み上げ。
- 初級で1マス進むたび: 通過後の数を「いち/に/…」ではなく **到達した数**を数える(例: 9,10,11…)か、進んだ歩数を数えるかは実装時に1つに決める(初期は「進んだ歩数」を数え、量の実感を優先)。
- 正解時: 「<goal>! やったね!」。不正解時: やさしい促し(「もういちど、かぞえてみよう」)。
- 音声 OFF 設定時は全てスキップ。

## データ永続化
- `localStorage` のみ使用。保存項目: 編成長(累計正解数)、選択中の足す数、モード、音声 ON/OFF。

## エラー / エッジケース
- SpeechSynthesis 非対応・音声未ロード時: 音声なしで動作継続(無音フォールバック)。
- 上級モードでの誤タップ連打: アニメーション中は入力をロックして二重発火を防ぐ。
- アニメーション中の「すすむ」連打: 同上、進行中はボタン無効化。
- start+addend が上限超過にならないよう出題側で保証(UI 側でのはみ出し描画を防ぐ)。

## テスト方針
- `board.js`: 数値↔(行,列)変換、折り返し経路の算出を入出力で検証(例: path(8,15) === [9,10,11,12,13,14,15])。
- `problem.js`: 生成された問題が常に `start+addend≤30`・`addend∈[1,9]`・おまかせ時の分布を満たすか。
- 軽量に保つため、ブラウザ不要の純粋関数テスト(Node で実行 or 簡易アサーション）。UI は手動確認。

## 検証(動作確認の手順)
1. ローカルで簡易サーバを起動(例: `python3 -m http.server`)し、PC ブラウザで開く。
   - Service Worker は `file://` では動かないため http で確認。
2. 初級モードで「8 たす 7」を出し、「すすむ」を7回押して 10→11 の折り返しを目視確認、15 で正解演出が出ること。
3. 上級モードで正解マス/不正解マスをタップし、判定とフィードバックを確認。
4. 設定で足す数を +7 に固定→+7 ばかり出ること、「おまかせ」で混ざること。
5. 正解ごとに「きみのでんしゃ」が伸び、リロード後も編成が保持される(localStorage)こと。
6. 音声 ON で読み上げ・数え上げ・ほめ言葉が出る/OFF で無音になること。
7. iPad 実機の Safari で開き、共有→「ホーム画面に追加」→機内モードで起動し、オフライン動作を確認。
8. 純粋関数テストを実行して board/problem のロジックが緑であること。
